# API Playground

`sparkify` v0.1.0 uses Stoplight Elements for interactive API pages.

Default API model is endpoint pages under `/api-reference/*` with a generated API intro.
Single-page playground mode remains available via `api.mode: "single-page"`.

## Support

- API key auth
- Bearer auth
- Basic auth
- OAuth2 browser-safe flows:
  - Authorization Code + PKCE
  - Device Code

## Static hosting constraints

On GitHub Pages, Try It requests are browser-side:

- API must allow CORS, or
- configure `playground.tryItCorsProxy`.

## Config example

```json
{
  "api": {
    "mode": "endpoint-pages",
    "apiRoot": "/api-reference"
  },
  "playground": {
    "provider": "stoplight",
    "tryItCorsProxy": ""
  }
}
```
